# 🚀 Laravel Справочник для начинающих

> **Важно:** Основная работа ведется в папке `resources` - все файлы и страницы создаются там, а затем автоматически переносятся в `public` после выполнения команды `php artisan vite`

---

## Установка и настройка проекта

### Требования
- PHP >= 8.2
- Composer
- Node.js и npm
- Git

### Шаги для установки

1. **Запустите автоматическую установку**
   ```bash
   chmod +x setup.sh
   ./setup.sh
   ```

### Быстрый запуск разработки

Для автоматического запуска всех необходимых процессов выполните команды из `package.json`:

```bash
# Запустить в 3 отдельных терминалах:
npm run 1  # Компиляция SCSS
npm run 2  # Запуск Laravel сервера
npm run 3  # Сборка Vite с отслеживанием изменений
```

После этого сайт будет доступен по адресу: **http://127.0.0.1:8000/**

## 📁 Создание страниц

### Добавление новой страницы

1. Создайте файл в `resources/views` с расширением `.blade.php`
2. Пример: для страницы "helloworld" создайте файл `helloworld.blade.php`

### Подключение маршрутов

В файле `routes/web.php` добавьте:

```php
Route::get('/ПУТЬ', function () {
    return view('ИМЯ_ФАЙЛА'); //тут имя файла - это helloworld (имя = все до .blade.php)
});
```

**Пример:**

```php
Route::get('/profile', function () {
    return view('profile'); //profile.blade.php
});
```

### Именованные маршруты

Для удобства можно добавить имя маршрута:

```php
Route::get('/profile', function () {
    return view('profile');
})->name('profile');
```
эти именовые маршруты удобны в том, что на странице потом можно просто вызвать маршрут написал @route('тут имя этого маршрута') бзе написания в ручную полного путя - удобно

## 📂 Работа с папками

Если страница находится в подпапке, используйте `/` или `.`:

```php
return view('ПАПКА/ИМЯ_ФАЙЛА');
// или
return view('ПАПКА.ИМЯ_ФАЙЛА');
```

## 🏗️ Создание страницы готовой в 1 действие

Blade - это система шаблонов с директивами `@`, позволяющая работать с PHP в HTML без использования `<?`

На новой страницу вставьте - это:

```blade
<!-- это html шаблон вставляем -->
@extends('componet/shablon')

<!-- HEAD ADD CONTENT -->
@section('title', 'Название страницы TITLE')
@section('description','Описание страницы DESCRIPTION')

<!-- BODY CONTENT -->
@section('content')

<!-- тут пишем наш html будто в <body> все стили и доп параметры css заранее стоят -->

@endsection
```

## 🎮 Контроллеры BACKEND

Контроллеры - это файлы с функциями, которые выполняются перед отображением страницы.

**Порядок выполнения:**

```
URL запрос → Функция в контроллере → HTML-страница
```

### Создание контроллера

```bash
php artisan make:controller ИмяКонтроллера
```

### Пример использования

```php
// routes/web.php
Route::get('/user/{id}', [UserController::class, 'show']);

// app/Http/Controllers/UserController.php
public function show($id) {
    $user = User::find($id);
    return view('user.profile', ['user' => $user]);
}
```

## 🛠️ Что нужно с самого начала!

Чтобы самим не запускать команды для просмотра сайта и работы с ним, заходим в файл package.json и находим scripts: {} там будут циферки 1, 2, 3. У вас в редакторе есть кнопка над ней запустить и запускаем по очереди все 3 и готово, работайте на здоровье все стили автоматически обновляются, а сайт работает на адресе: http://127.0.0.1:8000/

у нас стили используются в файле resources/css/app.scss - это такой быстрый css в котором куча сокращенныйх способов писать css без грязи. Фронтендерам нужно с ним поигратся, чтобы быстренько освоить.
